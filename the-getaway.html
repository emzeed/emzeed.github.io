THE GETAWAY

The GetAway is a booking/accomodation app for a fictional resort called The GetAway. The app was created for my Web & UX Design Diploma.

For this project, we were tasked to create a single page app using JavaScript that allowed users to book accomodation.

The Goal

We had to create a single page app using Vanilla JavaScript. Users had to be able to book a date, a type of accomodation that was dependant on the number of guests, and food from a menu. The app also had to include a JavaScript widget.

Concept Design

The App

I wanted the design to be very simple and straightforward to use because the target audience ranged from people who would be tech savy and those who weren’t. By creating an isolated section for each task, I felt users would find the app less overwhelming and confusing.

Visual Design

The fictional resort is a tree house accomodation service in the forests around New Zealand. I wanted the app to look very calm and nature-like to go with the theme of the resort.

Original Design

I used unsplash.com to find images for the app. I then based the colour scheme off of these images. Originally, the landing page was designed to have a carousel of sliding images to give the app more life.

Because of time restraints, I didn’t develop the design much further. However, I have since updated the design which I have included at the end.

Final Design

Visual

I went with the original design, adding some minor adjustments to work with the widget and functionality. I chose black and white to go with the green because it was simple, and made the information easily readable against the background.

Functionality

Each task for the user has it’s own page, and once the task is appropriatly completed, the user can move on to the next. The app is designed so that users can only book what is available to them.

Log In

Before beginning the booking process, users must sign up or log in. Only after the log in details are inputed does the app allow the user to move onto the next page.

Choosing Accomodation

As a requirement for the project, only some accomodation types were available depending on the number of guests.
In this app, accomodation was disabled until the user selected the number of people in their party. Afterwards, the available accomodation became inabled.

Choosing Dates

I added a calendar widget that allowed me to select start and end dates. Users were provided with the minimum and maximum number of nights available depending on their accomodation and then were able to select the dates. I added a ‘confirm dates’ button which checks that the number of nights selected is accurate, and if they are, then users can move forward.

Book Meals

Users were first given the option to select a dinner service or not. If they chose to, they were then directed to select a night from their stay, followed by a meal for each guest.
